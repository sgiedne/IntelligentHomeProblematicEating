What we do

We're working on building a system that is capable of recognizing patterns of normal and abnormal user behavior with respect to food habits within user’s homes. Our intention is to detect abnormal eating in “real-enough time” such that an intervention can be presented before or right after the behavior takes place. We will detect the user's actions and habits using sensors placed strategically in certain rooms (places of eating) and in food and food-related item storage locations.

End of Week 6

    Sensors Samsung SmartThings correctly detect users’ actions and using the API, we established a connection to the backend and sensors are currently posting data to the server in real time. Estimotes correctly detect users’ actions and data is being posted to the server as well. Various bug fixes were also implemented.
    Demo Our first demo has been planned and it will include a video of the sensors working in-home as well as a live demo in-class.
    Next steps Over the course of the next few days, we will refine the granularity of data we are collecting for analysis.

End of Week 7

    Paper Draft An initial draft of the paper on Detection of Problematic Eating was completed. Additional research and model development must be done in order to fine tune and complete the paper
    Things to think about We’ve realized that what we should be creating is an algorithm that can be personalized and can learn the overeating behaviors of a single person. This will not change how we execute the study, but it will change how we process it later on.
    As there are different types of overeating behaviors that can be exhibited by a single person, we need to think about how our algorithm can learn and differentiate between these different types of patterns.

End of Week 8

    Algorithms We're looking into possible clustering algoritms that can be used to segregate different patterns of overeating after data collection.
    Data Collection We have scripted patterns of overeating and collecting data using the predefined sripts. We have scaled down a day from 24 hours to half an hour. We are currenly planning to collect data for a period of 21 days
    Things to think about Since the number of overeating patterns can vary between different users, we cannot employ K Means clustering
    Next steps Begin data collection. Also read up literature on different types of clustering algorithms

End of Week 9

    Implementation We've decided to go with Mean Shift Clustering(MSC) to cluster input data. We have split the data with a window size of 3 hours. These 3 hour samples will be fed into the MSC algorithm. From the resulting clusters, we will target those clusters which have the largest number of overeating episodes within them and run each of these clusters through the Longest Common Subsequence(LCS) algorithm to abstract out patterns of overeating.
    Things to think about We must have the machine maintain state and remember the identified patterns of overeating. The system must also be able to provide a suitable form of intervetion to the user to prevent the user from overeating.
    Next steps Improve/update our implementation process and continue working on the paper.
